## 项目说明

这是一个用于在 Cursor 中实践 **Vibe Coding 氛围编程规范工作流** 的技能项目。

本项目在 `.cursor/skills/vibe-coding-workflow/` 下提供了一个自定义 Skill，帮助你用 AI 以「需求 → 架构 → 代码 → 调试 → 迭代」的规范流程来完成开发任务。

---

## Skill：vibe-coding-workflow

- **名称**：`vibe-coding-workflow`
- **位置**：`.cursor/skills/vibe-coding-workflow/SKILL.md`
- **作用**：规范 AI 在整个开发过程中的协作方式，强制按照以下 5 个阶段来推进：
  - **Phase 1 需求描述**：从一句模糊想法，走到结构化、可执行的需求文档（澄清 → 技术选型 → 模板固化）；
  - **Phase 2 架构设计**：在写第一行代码前，先确定项目目录结构、模块职责、数据流向和接口约定；
  - **Phase 3 代码生成**：按模块逐步实现，确保每次只生成一个模块的代码，并与前期架构保持一致；
  - **Phase 4 调试优化**：出错时提供完整错误信息，AI 用四步循环（看错误 → 解释 → 步进操作 → 一句话总结）协助排查；
  - **Phase 5 持续迭代**：根据「新增功能 / 性能或体验优化 / 重构」三种场景，选择合适 Phase 重新进入循环。

## 在 Cursor 中如何使用该 Skill

- **自动触发的典型场景**：
  - 你在对话中提到：`vibe`、`氛围编程`、`规范工作流`、`Phase 1/2/3/4/5` 等关键词；
  - 你说类似的话：**“帮我按照规范工作流一步一步做这个项目”**。
  在这些情况下，助手会按照 `SKILL.md` 中定义的 5 个 Phase 来引导你推进。

- **建议的使用方式**：
  1. 在 Cursor 中打开你的项目（或本仓库）；
  2. 在聊天中直接描述你的想法，例如：
     - “我想按 Vibe Coding Workflow 做一个 XXXX 工具，请从 Phase 1 帮我开始”；  
     - “这个项目已经写了一半，现在想按规范工作流整理一下架构”；  
  3. 跟随助手的提问和阶段划分，逐步完成：
     - 需求澄清与技术选型；
     - 架构设计与文档落盘；
     - 各模块代码生成；
     - 出错时的调试与总结；
     - 上线后的功能迭代或重构。

- **使用时的注意点**：
  - 每个阶段结束前，助手都会给出**检查清单**和**需要你确认的要点**，请尽量回答清楚再进入下一步；
  - 需求文档、架构文档、调试总结等建议你保存到仓库中（如 `docs/` 目录），后续新需求或重构时可以直接复用；
  - 如果你希望暂时跳过某个 Phase，可以在对话中明确说明（例如“这次先不做架构，直接从代码生成开始”），助手会在简要提示风险后尊重你的选择。
